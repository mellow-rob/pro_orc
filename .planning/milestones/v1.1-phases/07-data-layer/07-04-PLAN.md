  <behavior>
**ProjectScanner class:**
- Constructor takes `AppDatabase` instance (for config and project settings)
- `Future&lt;List&lt;ProjectModel&gt;&gt; scanAll({String? scanDirOverride})` — main entry point
- Internal `_FileCache` for mtime-based file read caching (per research Pattern 2)

**scanAll() flow:**
1. Read app config from DB: get `scanDir`, `ignoreListJson`, `gitBinaryPath`
   - If `scanDirOverride` provided, use it instead of DB value (for tests)
   - If scan dir is empty string and no override, throw `ScanDirectoryNotFoundError`
2. List direct children of scan dir via `Directory.list(recursive: false, followLinks: false)`
   - Filter to directories only (ignore files)
   - Skip hidden dirs (name starts with `.`)
   - Skip dirs matching ignore patterns from config (exact match or `*`-suffix pattern)
   - If scan dir doesn't exist, throw `ScanDirectoryNotFoundError`
3. For each discovered directory, concurrently:
   - Call `parseGsdData(projectPath)` for GSD + display name + description
   - Collect all paths for git reading
4. Call `readAllGitData(projectPaths, gitBinary: config.gitBinaryPath)` for all projects (chunked concurrently)
5. For each project, look up `getProjectSettings(folderId)` from DB for type
6. Assemble `ProjectModel` for each:
   - `folderId` = folder basename
   - `displayName` = GSD parse result displayName ?? folder basename
   - `path` = absolute path
   - `projectType` = DB settings value (null = unclassified). **Dual-storage deferral:** CONTEXT.md locks "dual storage with timestamp sync" for type, but read-side PROJECT.md type parsing is deferred because existing PROJECT.md files have no type field yet — you can't read what doesn't exist. The type field in PROJECT.md will be WRITTEN by the app when user sets a type (future UI phase). For Phase 7, DB-only read is correct. The `typeSetAt` timestamp field in project settings and the `resolveProjectType()` conflict-resolution pattern from research are prepared infrastructure for when dual-read activates
   - `description` = GSD parse result description
   - `gsd` = parsed GSD data (null if GsdData.isEmpty)
   - `git` = git data from readAllGitData result (null if GitData.isEmpty)
   - `hasParseError` = GSD parse result hasParseError
   - `isStale` = compute from git.lastCommitDate (>30 days) for git projects, or STATE.md mtime for non-git projects
7. Return `List&lt;ProjectModel&gt;` sorted by displayName

**ScanDirectoryNotFoundError:**
Custom exception class with `path` and `message` fields.

**Ignore pattern matching:**
- Exact match: `node_modules` matches `node_modules`
- Prefix `*`: `*.tmp` matches anything ending with `.tmp`
- Hidden dir check is separate (always applied, not configurable)

**Stale detection:**
- Git project: `git.lastCommitDate != null && DateTime.now().difference(git.lastCommitDate!) > Duration(days: 30)`
- Non-git project with .planning/: `FileStat.stat(STATE.md).modified` older than 30 days
- No signal: not stale (benefit of the doubt)

**Cases:**
- Empty scan dir → empty list (no error)
- Scan dir with mix of files and dirs → only dirs returned
- Hidden dirs skipped
- Ignore list patterns applied
- Non-existent scan dir → ScanDirectoryNotFoundError
- Project with .planning/ → full GSD data
- Project without .planning/ → null gsd, no error
- Project with git → full git data
- Project without git → null git, no error
- DB has type for project → projectType set
- DB has no type → projectType null
