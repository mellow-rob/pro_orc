  <behavior>
The parser reads three files from a project's `.planning/` directory concurrently and extracts structured data.

**Return type:** `GsdParseResult` (a local class in gsd_parser.dart):
```dart
class GsdParseResult {
  final GsdData gsd;           // Parsed GSD fields
  final String? displayName;   // H1 from PROJECT.md
  final String? description;   // from ## What This Is / ## Core Value
  final bool hasParseError;    // true if any file was malformed
}
```

**STATE.md extraction:**
- `**Phase:**` or `Phase:` → `currentPhase` (string, e.g. "3 of 5 (API Layer)")
- `**Status:**` or `Status:` → raw status string, then `deriveStatus()` normalizes to: research|planning|building|paused|done|archived
- `**Next Step:**` or `**Next Action:**` or `**Nächster Schritt:**` → `nextStep`

**ROADMAP.md extraction:**
- `- [x] NN-NN-PLAN` checkbox lines → count done plans
- `- [ ] NN-NN-PLAN` checkbox lines → count pending plans
- progress = done / (done + pending) * 100, rounded
- Also extract `phasesCompleted` / `phasesTotal` if phase checkboxes present

**PROJECT.md extraction:**
- H1 heading (`# Name`) → `displayName`
- `<!-- notion: URL -->` → `notionUrl`
- `## What This Is` / `## Core Value` / `## Kernwert` / `## Was ist das` section → `description` (first paragraph, 200 char max, strip bold markers)

**Cases:**
- No .planning/ → GsdParseResult with GsdData.empty, no error
- STATE.md missing, others present → partial result
- All present with valid data → full result
- Malformed content → partial result + hasParseError=true
- Bold format `**Phase:**` → extracts value
- Plain format `Phase:` → extracts value
- German field names → extracts value
  </behavior>

     - Next step: `^\*\*Next Action:\*\*\s*(.+)$`, `^\*\*Next Step:\*\*\s*(.+)$`, `^\*\*Nächster Schritt:\*\*\s*(.+)$`
