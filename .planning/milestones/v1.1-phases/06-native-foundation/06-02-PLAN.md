  <action>
    1. Create `lib/tray/tray_service.dart` — a class that implements TrayListener:
       - `init()`: Set icon to `assets/images/tray_icon.png`, set tooltip to "Pro Orc — 12 projects, 2 stale" (placeholder), create context menu with "Show/Hide Window" and "Quit" items, register as listener
       - `onTrayIconMouseDown()`: Toggle window visibility — if `windowManager.isVisible()` then `windowManager.hide()`, else `windowManager.show()` + `windowManager.focus()`
       - `onTrayIconRightMouseDown()`: Call `trayManager.popUpContextMenu()` explicitly
       - `onTrayMenuItemClick(MenuItem)`: Handle "show_hide" key (same toggle logic) and "quit" key (call `windowManager.destroy()`)
       - `dispose()`: Remove listener
       - IMPORTANT: Import `MenuItem` from `tray_manager`, NOT from `flutter/material.dart`. Use: `import 'package:flutter/material.dart' hide MenuItem;`

    2. Create `lib/window/window_geometry_service.dart`:
       - Keys: `window_x`, `window_y`, `window_w`, `window_h`
       - `save()`: Get position and size from `windowManager`, write all four to `SharedPreferences`
       - `restore()`: Read all four values. If any is null, return false (first launch). Otherwise, set size then position via windowManager. Return true.
       - After restoring, validate position is on-screen: if `x < -100 || y < -100`, fall back to `windowManager.center()` (simple off-screen guard for disconnected monitors)

    3. Update `lib/main.dart`:
       - Import WindowGeometryService
       - In `waitUntilReadyToShow` callback: create WindowGeometryService, call `restore()`. If it returns false (first launch), call `windowManager.center()`. Then `windowManager.show()` + `windowManager.focus()`.
       - Set `windowManager.setPreventClose(true)` inside the callback (after window is ready)
       - Do NOT initialize tray in main.dart — that happens in ShellScreen (Task 2) where the TrayListener mixin lives

    Note on tray initialization: `trayManager.ensureInitialized()` should be called in main() before runApp, but the TrayService.init() (which sets icon, menu, registers listener) happens in the ShellScreen widget's initState.
  </action>
