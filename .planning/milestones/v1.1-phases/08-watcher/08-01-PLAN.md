**2. Create WatcherService** at `pro_orc/lib/data/services/watcher_service.dart`:

A plain Dart class (no Riverpod dependency) that:
- Constructor takes `String rootDir` (the scan directory to watch, e.g. `~/project_orchestration/`)
- Exposes a `Stream<WatchEvent> get events` getter that:
  1. Creates a `DirectoryWatcher(rootDir)` on first access (lazy)
  2. Wraps the events stream with a `try/catch` in a `handleError` transformer — per locked decision, defensive code around watcher event processing regardless of watcher version (watcher#79)
  3. Applies `.debounce(const Duration(milliseconds: 350))` from `stream_transform` — trailing-edge, global (not per-project)
  4. Returns the debounced, error-guarded stream
- Has a `Future<void> dispose()` method that cancels internal subscriptions if any
- Has a `bool get isReady` getter that delegates to `DirectoryWatcher.isReady`

**Design decisions (Claude's Discretion):**
- Watch-Scope: Watch root scan directory (covers all three requirements — STATE.md changes, new project dirs, deletions)
- Debounce: 350ms fixed, trailing-edge, global batch (one timer for all events since we always do full scanAll)
- Start-Timing: Immediately on creation (resource cost of one FSEvents watcher is negligible)

**Defensive error handling pattern:**
```dart
Stream<WatchEvent> get events {
  _watcher ??= DirectoryWatcher(_rootDir);
  return _watcher!.events
      .handleError((error, stackTrace) {
        // Log but don't crash — defensive against watcher#79 and similar
        // In debug mode this prevents assertion crashes from propagating
      })
      .debounce(const Duration(milliseconds: 350));
}
```

**Imports:**
```dart

import 'package:stream_transform/stream_transform.dart';
```

**Test cases:**

1. **"emits event when file is created in watched directory"**
   - Create temp dir, start WatcherService on it
   - Wait for watcher to be ready (`watcher.isReady`)
   - Write a new file into the temp dir
   - Expect a WatchEvent to arrive within 2 seconds (use `expectLater` with `emitsThrough`)
   - Verify event.type is ChangeType.ADD or ChangeType.MODIFY (macOS may report either)

2. **"emits event when file is modified"**
   - Create temp dir with an existing file
   - Start WatcherService, wait for ready
   - Modify the file content
   - Expect a WatchEvent with ChangeType.MODIFY within 2 seconds

3. **"debounces rapid changes into single event"**
   - Create temp dir, start WatcherService
   - Wait for ready
   - Write 5 files rapidly (within 100ms)
   - Collect events over 1 second window
   - Assert that fewer events arrived than files written (debounce collapsed them)
   - This tests LIVE-01 requirement: "Multiple rapid file saves result in exactly one data refresh"

4. **"does not crash on directory events (watcher#79 defense)"**
   - Create temp dir, start WatcherService
   - Wait for ready
   - Create a subdirectory (triggers directory-level events on macOS)
   - Wait 1 second — no crash, no unhandled exception

**Test pattern:**
```dart
import 'dart:io';
import 'package:test/test.dart';
import 'package:pro_orc/data/services/watcher_service.dart';

void main() {
  late Directory tempDir;
  late WatcherService service;

  setUp(() {
    tempDir = Directory.systemTemp.createTempSync('watcher_test_');
    service = WatcherService(tempDir.path);
  });

  tearDown(() async {
    await service.dispose();
    if (tempDir.existsSync()) {
      tempDir.deleteSync(recursive: true);
    }
  });
  // ... tests
}
```

**Important:** File watcher tests are inherently timing-sensitive. Use generous timeouts (2-5 seconds) and `emitsThrough` matcher to tolerate OS-level delays. Mark the test group with `@Timeout(Duration(seconds: 30))` to avoid premature test runner timeout.
