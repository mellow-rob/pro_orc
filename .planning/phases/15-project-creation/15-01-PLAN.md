**Main function:**

```dart
Future<ProjectCreationResult> createProject({
  required String scanDir,
  required String folderName,        // Already kebab-case
  required String displayName,       // Original user input
  required String projectType,       // 'code' or 'research'
  bool gitInit = false,
  bool gsdSkeleton = false,
  bool claudeMd = false,
  String gitignoreTemplate = 'none', // 'flutter', 'nodejs', 'python', 'none'
}) async
```

**ProjectCreationResult class:**

```dart
class ProjectCreationResult {
  final String projectPath;
  final bool success;
  final List<String> warnings;  // Non-fatal errors (e.g. git init failed)

  const ProjectCreationResult({
    required this.projectPath,
    required this.success,
    this.warnings = const [],
  });
}
```

**Implementation steps (in order):**

1. **Create directory:** `Directory(path.join(scanDir, folderName)).create(recursive: true)`
   - If directory already exists, return failure result (should be prevented by dialog validation but safety check)

2. **GSD Skeleton** (if gsdSkeleton=true):
   - Create `.planning/` subdirectory
   - Create `PROJECT.md`:
     ```
     # {displayName}

     ## Beschreibung

     [Projektbeschreibung hier einfuegen]

     ## Status

     Neues Projekt, noch nicht gestartet.
     ```
   - Create `STATE.md`:
     ```
     # Project State

     ## Current Position

     Phase: -
     Status: Nicht gestartet
     ```
   - Create `ROADMAP.md`:
     ```
     # Roadmap: {displayName}

     ## Phases

     [Phasen hier definieren]
     ```
   - Create `REQUIREMENTS.md`:
     ```
     # Requirements: {displayName}

     ## Requirements

     [Anforderungen hier definieren]
     ```
   - All files in `.planning/` subdirectory of the project

3. **CLAUDE.md** (if claudeMd=true):
   - Create `CLAUDE.md` in project root:
     ```
     # CLAUDE.md

     ## Project Overview

     **{displayName}** — [Projektbeschreibung hier einfuegen]

     ## Build & Run Commands

     ```bash
     # [Build-Befehle hier einfuegen]
     ```

     ## Architecture

     [Architektur hier beschreiben]

     ## Conventions

     [Konventionen hier definieren]
     ```

4. **.gitignore** (if gitignoreTemplate != 'none'):
   - Create `.gitignore` in project root with minimal template (10-20 lines per user decision):

   **Flutter template:**
   ```
   .dart_tool/
   .packages
   build/
   .flutter-plugins
   .flutter-plugins-dependencies
   *.iml
   .idea/
   .vscode/
   *.lock
   .DS_Store
   ```

   **Node.js template:**
   ```
   node_modules/
   dist/
   build/
   .env
   .env.local
   *.log
   .DS_Store
   .vscode/
   .idea/
   coverage/
   ```

   **Python template:**
   ```
   __pycache__/
   *.py[cod]
   .env
   .venv/
   venv/
   dist/
   build/
   *.egg-info/
   .DS_Store
   .idea/
   .vscode/
   ```

5. **Research project files** (if projectType == 'research' and no GSD skeleton):
   - Create minimal `README.md`:
     ```
     # {displayName}

     [Projektbeschreibung hier einfuegen]
     ```

6. **git init + initial commit** (if gitInit=true):
   - Run `git init` via `Process.run('git', ['init'], workingDirectory: projectPath, runInShell: true)` — MUST use `runInShell: true` per project convention
   - Run `git add -A` then `git commit -m "Initial commit: {displayName}"`
   - If ANY git command fails, add warning to result but do NOT fail the entire operation (per user decision: "Ordner trotzdem erstellen, git-Fehler als Warnung anzeigen")
   - Use 5-second timeout per git command (same pattern as git_reader.dart)

**Import:** `import 'dart:io';` and `import 'package:path/path.dart' as path;`

**Error handling:** Return `ProjectCreationResult(success: false, ...)` only if directory creation itself fails. All other failures (git, file writes) are warnings.
