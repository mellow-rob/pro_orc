  <action>
**Hook — `hooks/useProjectEvents.ts`:**

1. Mark as `'use client'` at top.
2. Import `useEffect`, `useCallback` from React. Import `Project`, `SseEvent` from `@/lib/types`.
3. Export `useProjectEvents(onUpdate: (projectId: string, data: Project) => void)`:
   - In a `useEffect`, create `new EventSource('/api/events')`.
   - Set `source.onmessage`: parse `event.data` as `SseEvent`, filter for `type === 'project:updated'` with a truthy `projectId`, then `fetch('/api/projects/' + sseEvent.projectId)`. On success, parse JSON as `Project` and call `onUpdate(projectId, project)`. Wrap fetch in try/catch (ignore network errors — next event will retry).
   - Set `source.onerror` to no-op (EventSource reconnects automatically).
   - Return cleanup: `() => source.close()` — CRITICAL to prevent zombie connections.
   - Dependency array: `[onUpdate]` — caller MUST wrap `onUpdate` in `useCallback`.

**ProjectTabs Integration — `components/projectTabs.tsx`:**

1. Add imports: `useState`, `useCallback` from React, `useProjectEvents` from `@/hooks/useProjectEvents`, `Project`, `CodeProject`, `ResearchProject` from `@/lib/types`.
2. Inside `ProjectTabs`, add state: `const [liveData, setLiveData] = useState<Map<string, Project>>(new Map())`.
3. Create stable callback: `const handleUpdate = useCallback((projectId: string, project: Project) => { setLiveData(prev => new Map(prev).set(projectId, project)) }, [])`.
4. Call `useProjectEvents(handleUpdate)`.
5. Before the existing filtering logic, merge live data with props:
   ```
   const resolvedCode = codeProjects.map(p => (liveData.get(p.id) as CodeProject) ?? p)
   const resolvedResearch = researchProjects.map(p => (liveData.get(p.id) as ResearchProject) ?? p)
   ```
6. Replace all references to `codeProjects` with `resolvedCode` and `researchProjects` with `resolvedResearch` in the filtering and rendering logic (the `isPrivate` filter and map calls).

The tab counts will automatically update because they derive from the resolved arrays. Cards will re-render with new data when the `liveData` Map is updated.

**Important:** The `onUpdate` callback uses `useCallback` with empty deps, making it referentially stable. This prevents the `useEffect` in `useProjectEvents` from re-running on every render and creating new EventSource connections.
