**SSE Route ‚Äî `app/api/events/route.ts`:**

1. Add `export const dynamic = 'force-dynamic'` at top.
2. Export a `GET` function that accepts `NextRequest`.
3. Create a `ReadableStream` in the `start(controller)` callback:
   - Send initial ping: `controller.enqueue(encoder.encode('data: ' + JSON.stringify({ type: 'ping' }) + '\n\n'))`
   - Create a subscriber function that enqueues SSE-formatted events (wrap in try/catch for closed stream).
   - Import `watcherSubscribers` from `@/lib/watcher` and add the subscriber.
   - Listen for `request.signal` abort event ‚Äî on abort, delete subscriber from set and close controller.
4. Return `new Response(stream, { headers: { 'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache, no-transform', 'Connection': 'keep-alive', 'X-Accel-Buffering': 'no' } })`.

**Per-Project Route ‚Äî `app/api/projects/[id]/route.ts`:**

1. Add `export const dynamic = 'force-dynamic'`.
2. Export an `async GET` function with params typed as `{ params: Promise<{ id: string }> }` (Next.js 16 async params ‚Äî MUST await).
3. Call `scanProjectById(id)` (new function in scanner.ts).
4. If null, return `Response.json({ error: 'not found' }, { status: 404 })`.
5. Otherwise return `Response.json(project)`.

**Scanner Addition ‚Äî `lib/scanner.ts`:**

Add a new exported function `scanProjectById(id: string): Promise<Project | null>`:
1. Try direct path lookup: `path.join(PATHS.code, id)` ‚Äî check if directory exists with `fs.stat()`.
2. If not found, try `path.join(PATHS.research, id)`.
3. If neither exists, fall back to scanning both roots via the existing `scanDir()` and filter by matching `id`.
4. Once the project path is found, determine its type (code vs research) by checking which root it's under.
5. For code projects, enrich with git data via `getGitData()`. For research, return as-is.
6. Return the typed `Project` or `null`.

Note: `scanDir` is currently a private function in scanner.ts. It can be reused internally by `scanProjectById` since both are in the same module.
