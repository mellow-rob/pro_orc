  <action>
    1. Add `encrypt` package to pubspec.yaml dependencies (for AES encryption of API key):
       ```yaml
       encrypt: ^5.0.3
       ```
       Run `flutter pub get` from pro_orc/.

    2. Add two new columns to `app_config_table.dart`:
       ```dart
       TextColumn get notionApiKey => text().withDefault(const Constant(''))();
       TextColumn get notionParentPageId => text().withDefault(const Constant(''))();
       ```

    3. Create `pro_orc/lib/data/services/notion_crypto.dart` — top-level functions (consistent with memory_reader/git_reader pattern):
       - `String encryptNotionKey(String plainKey)` — AES-CBC encrypt with a static 32-byte app key, return base64-encoded ciphertext. If plainKey is empty, return empty string.
       - `String decryptNotionKey(String encryptedKey)` — reverse of above. If encryptedKey is empty, return empty string. Wrap in try-catch, return empty string on decryption failure.
       - Use a static final `_appKey` (32 bytes) and static final `_iv` (16 bytes) derived from a hardcoded app-specific string. This is NOT cryptographic security against sophisticated attacks — it prevents the API key from being visible as plaintext in the SQLite file.

    4. Update `app_database.dart`:
       - Bump `schemaVersion` from 2 to 3.
       - Add migration in `onUpgrade`: if from < 3, add both new columns via `m.addColumn(appConfigTable, appConfigTable.notionApiKey)` and `m.addColumn(appConfigTable, appConfigTable.notionParentPageId)`.
       - Add helper method `Future<void> updateNotionConfig({String? notionApiKey, String? notionParentPageId})` following the same pattern as existing `updateConfig()`. This method should accept ALREADY-ENCRYPTED values for notionApiKey (encryption happens in the UI layer before calling this).
       - The existing `getConfig()` already returns the full row including new columns, so no changes needed for reading.

    5. Run `dart run build_runner build --delete-conflicting-outputs` from pro_orc/ to regenerate `app_database.g.dart`.

    6. Create `pro_orc/test/data/notion_crypto_test.dart`:
       - Test: encrypt then decrypt round-trip returns original string
       - Test: empty string encrypt returns empty string
       - Test: encrypted output is NOT equal to plaintext input
       - Test: decrypt of garbage string returns empty string (graceful failure)
       Run `flutter test test/data/notion_crypto_test.dart` — all tests must pass.
  </action>

       ```dart
       final _notionPageIdController = TextEditingController();
       ```

    3. Dispose the new controllers in `dispose()`.

    4. In `_loadSettings()`, after loading existing config:
       ```dart
       _notionApiKey = decryptNotionKey(config.notionApiKey);
       _notionKeyController.text = _notionApiKey;
       _notionParentPageId = config.notionParentPageId;
       _notionPageIdController.text = config.notionParentPageId;
       ```

    5. Add save methods:
       ```dart
       Future<void> _saveNotionApiKey() async {
         final value = _notionKeyController.text.trim();
         if (value != _notionApiKey) {
           setState(() => _notionApiKey = value);
           final db = ref.read(appDatabaseProvider);
           await db.updateNotionConfig(notionApiKey: encryptNotionKey(value));
         }
       }

       Future<void> _saveNotionParentPageId() async {
         final value = _notionPageIdController.text.trim();
         if (value != _notionParentPageId) {
           setState(() => _notionParentPageId = value);
           final db = ref.read(appDatabaseProvider);
           await db.updateNotionConfig(notionParentPageId: value);
         }
       }
       ```
